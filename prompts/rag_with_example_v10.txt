STRUCTURE EXCEPTION RULE (ITEMS) (CRITICAL)

items is controlled ONLY by page_role.

If page_role = "detail":
- items MUST exist
- items MUST be an array
- if no rows exist → "items": []

If page_role = "cover", "summary", or "reply":
- omit the items key entirely


PAGE ROLE RULES

"detail" → repeating row items  
"summary" → totals / tax sections  
"cover" → company info 중심  
"reply" → fallback


COVER PAGE RULE (CRITICAL)

If page_role = "cover":

- prioritize extracting:
  recipient, issuer, target_period, closing_date, totals

- items must NOT exist

- NEVER summarize structured fields into plain text

- preserve full object structure even if partially null


COMPANY BLOCK MAPPING RULE (CRITICAL)

Before extracting recipient and issuer, group OCR lines into semantic company blocks.

Recipient:
- the block associated with the company line followed by "御中"

Issuer:
- the block associated with "担当部署", "電話", "FAX", or "振込先銀行"

Field assignment must be block-consistent


TOTALS STRUCTURE RULE (CRITICAL)

totals must preserve the nested object hierarchy shown in REFERENCE_JSON.

Never flatten totals into a string.


ROW INTERPRETATION RULE (CRITICAL)

- Do NOT blindly treat each OCR line as one item
- Use REFERENCE_JSON to determine:
  - how many rows should exist
  - how rows are grouped
  - how multiple OCR rows combine into one item

- Multiple OCR rows may map to ONE item
- A single OCR row may map to ONE item

- Especially:
  - combine rows with same 商品コード / JANコード
  - merge rows when they represent:
    - 金額条件 + 数量条件
    - multiple rebate conditions
  - map into:
    条件 / 条件2 / 金額 / 金額2
