代表スーパー 팝업 모달을 완전히 뜯어고쳐볼까 해
지금 주요한 문제는 RAG를 최우선으로 매핑을 하고 있는데
이게 엉망진창인 경우가 많다는 거야
그렇다고 엑셀을 우선으로 하면 사용자가 매핑을 수정하고 학습 요청을 할 때 그 다음부터는 RAG를 참조하지 않게 되고
그래서 내가 생각한 거는 최초 분석 결과가 아니라  1, 2차 검토가 끝난 항목에 대해서만 RAG 정답지로 사용하는 거야..
물론 사용자가 학습요청할 떄는 검토 여부와 상관없이 RAG 정답지로 사용하고(지금처럼)
그리고 우선순위는... 어떻게 해야할지 잘 모르겠어... 너가 생각해서 답해줘

**목표**
청구서에 있는 得意先名를 보고 소매처코드와 판매처코드를 제대로!!! 매핑하는 것
그리고 매핑한 결과를 바탕으로 정확한 마스터를 만들어내는 것
현재 원천 참조하는 파일의 정합성이 낮은 편임....

중요한 것은 사용자는 코드에 대한 이해도가 없다는 것
오로지 이름을 기준으로 매핑을 시도하기 때문에 코드만 보고서는 어떤 판매처/소매처인지 사용자가 모른다는 것을 전제로 할 것


원천 참조하는 파일
retail_master - 소매처코드, 소매처명
domae_retail_master - 소매처코드, 도매소매처코드
dist_master - 판매처코드, 판매처명
dist_retail_master - 판매처코드, 소매처코드
그리고...RAG 벡터DB

**화면 구성 자체는 조합이 아니라 판매처코드 따로, 소매처코드 따로!!**
왜냐하면,,, dist_retail_master 자체의 정합성이 확보되지 않았음

**소매처코드 우선순위 로직**
1. form_type이 1번 양식지이고 得意先コード가 있고 NULL이 아닌 경우
得意先コード를 domae_retail_master의 도매소매처코드와 비교해서 일치하는 경우 해당 행의 소매처코드를 반환
-> 정확히 일치하는 경우만 볼 테니 1행을 반환

2. 그 외의 경우에는 모두 得意先名를 retail_master의 소매처명과 비교해서 유사한 경우 해당 행의 소매처코드를 반환
-> 이름 유사도를 계산하는 것이기 때문에 유사도가 높은 순으로 최대 5개 반환

**판매처코드 우선순위 로직**
판매처의 근거가 되는 것은 청구서 페이지 자체에 있지 않고 다른 페이지에 있기 떄문에
판매처코드는 소매처코드가 확정되고 나서 dist_retail_master를 활용해 매핑한다
-> 이 조합 자체가 틀릴 수도 있기 때문에 사용자가 직접 **판매처명**을 검색해서 선택할 수 있게끔 해야 한다

**매핑 화면**
소매처명 : 得意先名
소매처코드 : 1순위에 있는 소매처코드
   소매처코드, 소매처명 추천 목록...

2. ..
3. ..

검색창(소매처명을 입력하면 retail_master에서 검색한 결과를 뿌려줌)
...        適用(버튼 클릭 시 위에 있는 소매처코드란에 값이 채워짐)
...

판매처코드: 1순위에 있는 판매처코드
    판매처코드 판매처명 추천 목록
1. ..
2. ..
3. ..

검색창(판매처명을 입력하면 dist_master에서 검색한 결과를 뿌려줌)
...        適用(버튼 클릭 시 위에 있는 판매처코드란에 값이 채워짐)
...

**최종 결과**
하나의 행은 판매처코드, 판매처명, 소매처코드, 소매처명을 갖게 되며
이 때 소매처명은 원천 파일에 있는 소매처명이 아닌 得意先名을 사용한다.
